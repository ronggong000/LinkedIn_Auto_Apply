# PRD（中文）— Seek 申请加速工具（MVP）

## 1. 背景与问题
用户在 Seek 申请岗位时，经常需要重复执行以下工作：
1. 上传简历并针对岗位调整内容；
2. 撰写/改写 cover letter；
3. 反复填写技能、薪资、签证、入职时间等筛选题。

这些重复步骤导致申请效率低、心理负担高、且容易出现信息不一致。

## 2. 产品目标
构建一个“AI + 自动填表”的申请加速工具，帮助用户在**保持真实性与可控性**前提下，显著减少重复劳动。

### 2.1 成功指标（MVP）
- 单个岗位平均准备时间减少 50% 以上；
- 简历/cover letter 生成后一次可用率 > 70%；
- 自动填表字段正确率 > 90%（常见字段）；
- 用户对“最终人工确认”流程满意度 > 80%。

## 3. 目标用户
- 频繁投递岗位的求职者；
- 需要为不同岗位快速定制申请材料的候选人；
- 对自动化有兴趣但需要可控、可审阅流程的用户。

## 4. 产品范围

### 4.1 MVP 必做
1. **用户资料中心**：
   - 个人信息（姓名、联系方式、签证、薪资期望、可入职时间等）；
   - 技能（名称、熟练度、年限）；
   - 工作经历/项目经历结构化信息。
2. **JD 解析**：
   - 从职位描述抽取职责、必备技能、加分项、关键词。
3. **文档生成**：
   - 按岗位输出定制 Resume 与 Cover Letter（PDF / DOCX）。
4. **Seek 表单辅助填写**：
   - 自动填写常见问题；
   - 自动上传生成文档。
5. **提交前人工确认**：
   - 用户可预览并编辑答案，确认后再提交。

### 4.2 暂不纳入 MVP
- 无人值守全自动投递；
- 多平台统一投递（LinkedIn/Indeed 等）；
- 全自动跟进邮件与面试排程。

## 5. 核心流程
1. 用户录入/导入个人资料与基础简历；
2. 用户打开 Seek 岗位页面并触发插件；
3. 插件提取 JD，发送后端解析；
4. AI 生成岗位定制简历与 cover letter；
5. 插件自动填充常见问题并上传文件；
6. 用户在提交前审阅、修改、确认提交；
7. 系统记录申请结果与文档版本。

## 6. 功能需求（MVP）

### 6.1 资料管理
- 支持表单录入与简历解析导入；
- 支持多个求职偏好配置（行业/地区/薪资范围）。

### 6.2 AI 生成
- 输入：用户结构化信息 + JD 结构化结果；
- 输出：
  - 简历摘要与经历 bullet 定制版本；
  - cover letter 正文；
  - 申请问题建议答案（含置信度）。
- 约束：严禁编造事实。

### 6.3 自动填表
- 字段映射：work rights、visa sponsorship、expected salary、notice period 等；
- 支持失败回退与人工补填。

### 6.4 申请记录
- 记录岗位、公司、投递时间、使用文档版本、问题答案、状态。

## 7. 非功能需求
- 安全：敏感信息加密存储、传输 HTTPS；
- 可观测：记录失败日志与字段匹配失败原因；
- 性能：生成任务异步处理，平均 20 秒内返回草稿；
- 可维护：字段映射与提示词模板可配置。

## 8. 风险与策略
1. **平台条款风险**：优先半自动流程，保留人工确认；
2. **幻觉风险**：规则引擎 + 结构化校验 + 必填真实性检查；
3. **页面变动风险**：字段选择器版本化 + fallback 匹配。

## 9. 里程碑
- Week 1：PRD + 数据模型 + Prompt 模板冻结；
- Week 2：AI 生成链路打通（文本输入输出）；
- Week 3：Chrome 插件自动填表 MVP；
- Week 4：稳定性、日志、版本管理与 UX 优化。
